GIS Data Visualizer

At Geomatix, most of our deliverables are incredibly cumbersome and unfamiliar filetypes. A typical project ends up with a raster map greater than 3GB and a .las file point-cloud of a similar size. For many of our clients, particularly those on the management teams, using these files is near impossible. To make our work-products more accessible, I designed, developed, deployed, and maintained a customer-facing website where we could host these files and share them with our clients in a way that didn't requires advanced software or know-how.

This project is a large django app, deployed to Heroku. Files are hosted on AWS S3 and served over AWS CloudFront. Clients are invited to a website that is not accessible to the public where they may view, manipulate, and download their data. The open source projects OpenLayers and Potree have been used as the primary visualization tools.

A large part of the project is built to pre-process and upload the many thousands of files required for a single dataset. While the project was originally designed to handle this pre-processing in a separate celery queue, cost concerns have dictated that we perform data-preprocessing locally and upload the final files directly to S3.

While the source code for this is largely owned by my employer, they have allowed me to share a link to a set of sample dataset in my portfolio.

Minecraft Server Hosting

This is a django app adapted from an opensource flask app. Hosted on my personal website, it allows me and my friends access to a tool that can affordably manage and host new or stored minecraft instances.

This app is capable of provisioning new EC2 instances, setting them up with a standard AMI, installing and configuring the newest version of minecraft, and returning a public IP address for client connection. It also allows users to spin-down instances at will, and automatically turns off running EC2 instances after a set amount of downtime.
