GIS Data Visualizer

At my former employer, Geomatix, deliverables were typically large and an in unfamiliar format for most users. To address this, I built out a custom client portal to help clients visualize and interact with these files. A typical project would result in a large .TIFF raster map, a .LAS point-cloud, and other smaller CAD files. For users unfamiliar with CAD or GIS software, opening these 5+ GB files was not a realistic option, so I converted our download portal into a GIS data visualization tool.

This project was built as a large django app, deployed to Heroku. Files were hosted on AWS S3 and served over AWS CloudFront. Clients are invited to a private website where they may view, manipulate, and download their data. The open source projects OpenLayers and Potree have been used as the primary visualization tools.

The web interface is supported on the backend by a robust codebase designed to process, tag and upload files to the website. A single GeoTiff map can generate many thousands of files when processed and uploaded.

The source code for this is owned by my former employer, but they have allowed me to share a link to a set of sample dataset for my portfolio. Please note that this project is no longer hosted or maintained by me, and may or may not work as it was originally designed.
